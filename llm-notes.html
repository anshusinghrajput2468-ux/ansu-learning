Ch 01

AI & LLM क्या है? — Introduction

AI = Artificial Intelligence — machines को intelligent बनाना। LLM = Large Language Model — AI का एक type जो text समझता और generate करता है।

AI के Types:

  • Machine Learning (ML) — data से patterns सीखता है (recommendation, spam filter)
  • Deep Learning — neural networks use करता है (image recognition)
  • NLP (Natural Language Processing) — human language समझना (chatbots, translation)
  • LLM — बहुत बड़ा NLP model — ChatGPT, Gemini, Claude, LLaMA
  • Generative AI — नया content generate करना (text, image, code, video)

Popular LLMs:

Model Company Use
GPT-4 / ChatGPT OpenAI General purpose, coding, writing
Gemini Google DeepMind Multimodal (text + image + video)
Claude Anthropic Long context, reasoning, safety
LLaMA Meta Open-source, research
Mistral Mistral AI Efficient, open-weight
Ch 02

LLMs कैसे काम करते हैं?

LLM basically next word predict करता है! बहुत बड़ा data पढ़कर pattern सीखता है:

🧠 Simple Explanation
// LLM Training Process (simplified):

// Step 1: Pre-training — internet का data पढ़ो
//   Books + Wikipedia + Code + Websites = Trillions of words

// Step 2: Pattern Learning
//   Input:  "The cat sat on the ___"
//   Model:  "mat" (highest probability: 73%)
//           "chair" (12%), "floor" (8%)...

// Step 3: Fine-tuning — specific tasks के लिए train
//   RLHF = Reinforcement Learning from Human Feedback
//   Humans बताते हैं कि कौन सा response better है

// Step 4: Inference — user query answer
"JavaScript में array reverse कैसे करें?"
// → Model word-by-word response generate करता है

Key Concepts:

• Parameters = model की "memory" (GPT-4 = ~1.8 trillion parameters)

• Transformer = architecture जो LLMs use करते हैं (Attention mechanism)

• Context Window = एक बार में कितना text process कर सकता है (tokens)

• Temperature = creativity level (0 = precise, 1 = creative)

Ch 03

Prompt Engineering — AI से सही काम लेना

Prompt Engineering = AI को सही instructions देना ताकि बेहतर output मिले।

📝 Prompt Techniques
// ❌ Bad Prompt:
"code लिखो"

// ✅ Good Prompt:
"JavaScript में एक function लिखो जो array of numbers
 लेता है और even numbers filter करके sum return करता है।
 Arrow function use करो, comments दो, और edge cases
 handle करो (empty array, non-number values)।"

// ⭐ Techniques:

// 1. Role Prompting
"तुम एक senior React developer हो। मुझे बताओ..."

// 2. Few-Shot (examples दो)
"Input: 'hello' → Output: 'HELLO'
 Input: 'world' → Output: ??"

// 3. Chain of Thought (step-by-step सोचने कहो)
"Step by step सोचो और बताओ कि..."

// 4. Constraints (limits set करो)
"5 bullet points में, beginner level, Hindi में..."

// 5. Output Format specify करो
"JSON format में output दो:
 { name: string, skills: string[] }"

💡 Golden Rules: 1) Be specific (vague मत बोलो) 2) Context दो (कौन हो, क्या चाहिए) 3) Format बताओ (code, list, table) 4) Examples दो 5) Iterate करो (एक बार में perfect नहीं होगा)

Ch 04

AI APIs — Code से AI Call करना

🔌 OpenAI API — JavaScript
// npm install openai
import OpenAI from "openai";

const client = new OpenAI({
    apiKey: "your-api-key"  // ⚠️ Never hardcode in production!
});

async function askAI(question) {
    const response = await client.chat.completions.create({
        model: "gpt-4",
        messages: [
            { role: "system", content: "You are a helpful coding tutor." },
            { role: "user", content: question }
        ],
        temperature: 0.7,      // Creativity (0-1)
        max_tokens: 500        // Response limit
    });

    return response.choices[0].message.content;
}

askAI("Binary Search explain in Hindi")
    .then(answer => console.log(answer));
🔌 Google Gemini API
// npm install @google/generative-ai
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI("your-api-key");
const model = genAI.getGenerativeModel({ model: "gemini-pro" });

const result = await model.generateContent("Explain OOP in Hindi");
console.log(result.response.text());
Ch 05

Tokens & Parameters — Important Concepts

🎛️ Key Parameters
// Tokens = words/subwords (1 word ≈ 1-3 tokens)
"Hello, how are you?"  // = 6 tokens
"JavaScript"           // = 1-3 tokens

// Key API Parameters:
{
    temperature: 0.0,   // Deterministic (same input = same output)
    temperature: 0.7,   // Balanced (recommended)
    temperature: 1.0,   // Very creative (random)

    max_tokens: 500,    // Maximum response length
    top_p: 0.9,         // Nucleus sampling (diversity control)

    system: "...",       // System prompt (persona/rules)
    stop: ["###"],      // Stop generation at this string
}
Parameter Low Value High Value
Temperature Precise, factual Creative, varied
Max Tokens Short response Long & detailed
Top P Focused output Diverse output
Ch 06

RAG — Retrieval Augmented Generation

RAG = LLM को अपना custom data feed करना ताकि वो relevant answers दे:

📚 RAG Process
// RAG Pipeline:

// Step 1: Document Load
//   अपने PDFs, docs, websites load करो

// Step 2: Chunking
//   Documents को छोटे pieces (chunks) में तोड़ो

// Step 3: Embeddings
//   Chunks को vectors (numbers) में convert करो
//   Similar meaning = similar vectors

// Step 4: Vector Store
//   Vectors को database में store करो (Pinecone, Chroma)

// Step 5: Query
//   User: "Company की refund policy क्या है?"
//   → Query vector बनाओ
//   → Similar chunks ढूंढो (semantic search)
//   → Found chunks + user question → LLM
//   → LLM relevant answer generate करता है!

// ⭐ Use Cases:
//   - Company chatbot (trained on company docs)
//   - Personal knowledge base
//   - Code documentation Q&A

💡 RAG vs Fine-Tuning: RAG = apna data dynamically attach करो (flexible, updated)। Fine-tuning = model itself ko train करो (permanent, expensive)। ज़्यादातर cases में RAG enough है!

Ch 09

LLM Limitations — कहाँ गलती करता है?

  • Hallucination — confident होकर गलत info बताना (fake functions, wrong library names)
  • Outdated Info — training data का cutoff date होता है (latest info नहीं)
  • Math Weakness — complex calculations में गलती कर सकता है
  • Context Limit — बहुत लंबा code/text process नहीं कर पाता
  • Bias — training data में जो biases हैं वो output में आ सकते हैं
  • No Real Understanding — patterns follow करता है, truly "समझता" नहीं
  • Security — sensitive code/data share करने में risk

⚠️ Golden Rule: AI generated code को हमेशा review और test करो! Blindly copy-paste मत करो। AI = assistant, not replacement।

Ch 10

Ethics & Safety

  • Plagiarism — AI generated content को अपना मत बताओ, credits दो
  • Privacy — sensitive data (passwords, API keys) AI को मत दो
  • Verify — AI generated code/info को verify करो before using
  • Job Impact — AI jobs replace नहीं करता, AI-using developers replace करेंगे non-AI developers को
  • Deepfakes — AI generated fake content बनाना unethical है
  • Bias Awareness — AI output में bias check करो

💡 Mindset: AI एक power tool है — chainsaw की तरह। सही use = productive। गलत use = dangerous। Responsible use सीखो!

Ch 12

🎯 Interview Questions

Q1: LLM कैसे काम करता है?

→ Transformer architecture use करता है। Training data से patterns learn करता है। Inference time पर next token predict करता है।

Q2: RAG क्या है?

→ Retrieval-Augmented Generation। External documents से relevant info retrieve करके LLM को context देना ताकि accurate answers दे।

Q3: Temperature parameter क्या है?

→ Output randomness control करता है। 0 = deterministic/precise। 1 = creative/random। Most cases में 0.7 good है।

Q4: Hallucination क्या है?

→ LLM confidently गलत information generate करना। Solution: RAG, verification, fact-checking, lower temperature।

Q5: Fine-tuning vs RAG कब use करें?

→ Fine-tuning = style/behavior change (expensive, permanent)। RAG = knowledge add (cheap, flexible, updatable)। ज़्यादातर RAG enough।
