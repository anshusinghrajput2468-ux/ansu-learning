Ch 01

HTML क्या है? — Introduction

HTML का full form है HyperText Markup Language। यह web का foundation है — हर website HTML से बनती है।

Simple शब्दों में: HTML website का skeleton (ढांचा) है। जैसे शरीर की हड्डियां होती हैं, वैसे ही website का structure HTML से बनता है।

HTML क्यों ज़रूरी है?

  • हर website का base — बिना HTML के कोई webpage exist नहीं करता
  • Browser समझता है — Browser HTML पढ़कर text, images, links display करता है
  • सबसे आसान भाषा — कोई programming experience ज़रूरी नहीं
  • SEO का foundation — Google आपकी site को HTML structure से समझता है
  • CSS और JS का base — बिना HTML के CSS और JavaScript काम नहीं करती

💡 याद रखें: HTML एक programming language नहीं है — यह एक markup language है। इसमें कोई logic, loops, या conditions नहीं होते। बस structure define होता है।

Ch 02

Basic Structure — HTML का ढांचा

हर HTML file का एक fixed structure होता है। Browser को बताना पड़ता है — "यह एक HTML document है":

📄 index.html — Basic Structure
<!-- 1. Document Type Declaration -->
<!DOCTYPE html>

<!-- 2. HTML Root Element (lang = भाषा) -->
<html lang="hi">

<!-- 3. Head — Meta info (browser को) -->
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>मेरी पहली Website</title>
</head>

<!-- 4. Body — जो screen पर दिखता है -->
<body>
    <h1>नमस्ते दुनिया! 🌍</h1>
    <p>यह मेरी पहली website है।</p>
</body>

</html>

हर part का मतलब:

Part क्या करता है?
<!DOCTYPE html> Browser को बताता है कि यह HTML5 document है
<html> पूरे page का root element — सब कुछ इसके अंदर आता है
<head> Meta info — title, charset, CSS links (screen पर नहीं दिखता)
<body> जो content user को दिखता है — text, images, buttons सब यहाँ
Ch 03

Text Tags — Text को Format करना

HTML में text दिखाने के लिए अलग-अलग tags होते हैं:

📝 Text Tags Example
<!-- Headings: h1 सबसे बड़ा, h6 सबसे छोटा -->
<h1>Main Heading (सबसे important)</h1>
<h2>Sub Heading</h2>
<h3>Section Heading</h3>

<!-- Paragraph -->
<p>यह एक paragraph है। लंबा text लिखने के लिए।</p>

<!-- Text Formatting -->
<strong>Bold Text (important)</strong>
<em>Italic Text (emphasis)</em>
<mark>Highlighted Text</mark>
<del>Deleted/Strikethrough</del>
<small>Small Text</small>
<sub>Subscript (H₂O)</sub>
<sup>Superscript (x²)</sup>

<!-- Line break & Horizontal line -->
<br>    <!-- New line -->
<hr>    <!-- Horizontal line -->

⚡ Pro Tip: <b> और <strong> दोनों bold करते हैं, लेकिन <strong> SEO के लिए better है क्योंकि यह Google को बताता है कि text important है!

Ch 04

Links & Images — जोड़ना और दिखाना

🔗 Links (Anchor Tag)

🔗 Links Example
<!-- Basic link -->
<a href="https://google.com">Google पर जाएं</a>

<!-- New tab में खोलें -->
<a href="https://google.com" target="_blank">New Tab</a>

<!-- Same page का section (bookmark) -->
<a href="#contact">Contact Section</a>

<!-- Email link -->
<a href="mailto:anshu@email.com">Email भेजें</a>

🖼️ Images

🖼️ Images Example
<!-- Basic image -->
<img src="photo.jpg" alt="मेरी फोटो">

<!-- Size set करें -->
<img src="logo.png" alt="Logo" width="200" height="100">

<!-- Image with link -->
<a href="https://google.com">
    <img src="google-logo.png" alt="Google">
</a>

⚠️ Important: alt attribute हमेशा दें! यह image load न होने पर text दिखाता है, screen readers इसे पढ़ते हैं, और SEO के लिए ज़रूरी है।

Ch 06

Tables — टेबल बनाना

📊 Table Example
<table border="1">
    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>City</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Anshu</td>
            <td>20</td>
            <td>Delhi</td>
        </tr>
        <tr>
            <td>Priya</td>
            <td>22</td>
            <td>Mumbai</td>
        </tr>
    </tbody>
</table>
Tag काम
<table> Table शुरू करता है
<thead> Table का header section
<tbody> Table का body (data rows)
<tr> Table Row (एक पूरी line)
<th> Table Header cell (bold, centered)
<td> Table Data cell (normal data)
colspan="2" Cell को 2 columns में merge करें
rowspan="2" Cell को 2 rows में merge करें
Ch 07

Forms — User से Input लेना

Forms website का सबसे interactive part है — login, signup, contact, search — सब forms है!

📝 Complete Form Example
<form action="/submit" method="POST">

    <!-- Text Input -->
    <label for="name">नाम:</label>
    <input type="text" id="name" placeholder="अपना नाम लिखें" required>

    <!-- Email -->
    <input type="email" placeholder="email@example.com">

    <!-- Password -->
    <input type="password" placeholder="Password">

    <!-- Number -->
    <input type="number" min="1" max="100">

    <!-- Radio Buttons -->
    <input type="radio" name="gender" value="male"> Male
    <input type="radio" name="gender" value="female"> Female

    <!-- Checkbox -->
    <input type="checkbox"> I agree to terms

    <!-- Dropdown -->
    <select>
        <option>HTML</option>
        <option>CSS</option>
        <option>JavaScript</option>
    </select>

    <!-- Textarea -->
    <textarea rows="4" placeholder="Message लिखें..."></textarea>

    <!-- Submit Button -->
    <button type="submit">Submit करें</button>

</form>

💡 Tip: <label> में for attribute हमेशा दें — यह input के id से match होना चाहिए। इससे label click करने पर input focus होता है!

Ch 08

Semantic HTML — Meaningful Tags

Semantic tags browser और Google दोनों को बताते हैं कि content का मतलब क्या है:

🏗️ Semantic Structure
<header>     <!-- Page का top — logo, nav -->
    <nav>      <!-- Navigation links -->
        <ul>...</ul>
    </nav>
</header>

<main>       <!-- Primary content (एक ही page पर एक main) -->
    <section>  <!-- Thematic group (About, Services) -->
        <h2>...</h2>
    </section>

    <article>  <!-- Independent content (blog post) -->
        <h2>...</h2>
    </article>

    <aside>    <!-- Side content (sidebar, ads) -->
    </aside>
</main>

<footer>     <!-- Page का bottom — copyright, links -->
</footer>
Semantic Tag Non-Semantic फायदा
<header> <div class="header"> Google जानता है यह header है
<nav> <div class="nav"> Site structure Google समझता है
<main> <div class="content"> Primary content identify होता है
<article> <div class="post"> Screen readers navigate कर पाते हैं

Ch 11

🤖 AI / LLM Tips for HTML

Developer के तौर पर AI का सही इस्तेमाल करें, लेकिन AI की common गलतियों से भी बचें:

✅ AI से ये करवाएं:

• "Registration form का HTML structure बनाओ with all input types"

• "इस page को semantic HTML में convert करो"

• "Table का HTML generate करो with this data..."

• "मेरे HTML में accessibility issues बताओ"

⚠️ AI इन गलतियाँ कर सकता है:

• Heading hierarchy skip — AI कभी-कभी h1 के बाद सीधा h3 या h4 use कर देता है

• Alt text generic — AI "image" या "photo" जैसे vague alt text देता है

• Deprecated tags — AI कभी-कभी पुराने tags जैसे <center>, <font> use कर देता है

• Self-closing tags — AI <img/> लिख सकता है, HTML5 में / optional है

• Form validation missing — AI required, minlength जैसे attributes भूल सकता है

Ch 12

🎯 Interview Questions

Q1: HTML और XHTML में क्या फर्क है?

→ XHTML stricter है — सभी tags lowercase में, proper closing ज़रूरी। HTML5 ज़्यादा flexible है।

Q2: <div> और <span> में क्या फर्क है?

→ div = block element (पूरी width लेता है, new line)। span = inline element (text के बीच में)।

Q3: Semantic HTML Tags क्यों important हैं?

→ SEO improve होता है, accessibility बढ़ती है, code readable होता है, screen readers navigate कर पाते हैं।

Q4: <b> और <strong> में क्या अंतर है?

→ <b> = बस visual bold। <strong> = semantic importance (SEO benefit)।

Q5: localStorage, sessionStorage, और Cookies में फर्क?

→ localStorage = permanent, 5MB। sessionStorage = tab close पर delete। Cookies = server को भेजे जाते हैं, 4KB limit।

Q6: <section> और <div> कब use करें?

→ अगर content thematically related हो (heading हो) तो section। सिर्फ styling wrapper चाहिए तो div।
