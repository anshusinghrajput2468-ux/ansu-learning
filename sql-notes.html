Ch 01

SQL क्या है? — Introduction

SQL = Structured Query Language। यह database से बात करने की भाषा है — data store करना, निकालना, update करना, delete करना — सब SQL से होता है!

  • हर application का base — users, orders, products — सब data SQL database में रहता है
  • Backend ज़रूरी — Node.js, Java, Python — सब SQL use करते हैं
  • Interviews — Backend, Data Analyst, Full Stack — सबमें SQL पूछा जाता है
  • Universal — MySQL, PostgreSQL, SQLite, SQL Server — syntax लगभग same
  • Data Analysis — reports, dashboards, business intelligence

💡 SQL vs NoSQL: SQL = structured data (tables, rows) — MySQL, PostgreSQL। NoSQL = unstructured/flexible — MongoDB, Firebase। ज़्यादातर apps SQL use करते हैं!

Ch 02

Database & Tables बनाना

📦 Database & Table Creation
-- Database बनाना
CREATE DATABASE isha_school;
USE isha_school;

-- Table बनाना
CREATE TABLE students (
    id      INT PRIMARY KEY AUTO_INCREMENT,
    name    VARCHAR(100) NOT NULL,
    email   VARCHAR(150) UNIQUE,
    age     INT,
    city    VARCHAR(50) DEFAULT 'Unknown',
    created TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Table structure देखें
DESCRIBE students;
SHOW TABLES;
Data Type Use Example
INT Whole numbers age, id, quantity
VARCHAR(n) Variable text (max n chars) name, email
TEXT Long text description, bio
DECIMAL(p,s) Precise decimal price (10.99)
DATE Date only 2024-01-15
TIMESTAMP Date + Time 2024-01-15 10:30:00
BOOLEAN True/False is_active
Ch 03

SELECT — Data निकालना

🔍 SELECT Queries
-- सब columns
SELECT * FROM students;

-- Specific columns
SELECT name, email, city FROM students;

-- Alias (rename column)
SELECT name AS 'Student Name', age AS 'उम्र' FROM students;

-- Unique values only
SELECT DISTINCT city FROM students;

-- Limit results
SELECT * FROM students LIMIT 5;

-- Order by (sort)
SELECT * FROM students ORDER BY age DESC;
SELECT * FROM students ORDER BY name ASC;
Ch 04

WHERE — Filtering

🎯 WHERE Clause
-- Exact match
SELECT * FROM students WHERE city = 'Delhi';

-- Comparison
SELECT * FROM students WHERE age >= 18;
SELECT * FROM students WHERE age BETWEEN 18 AND 25;

-- Multiple conditions
SELECT * FROM students WHERE city = 'Delhi' AND age > 20;
SELECT * FROM students WHERE city = 'Delhi' OR city = 'Mumbai';

-- IN (list match)
SELECT * FROM students WHERE city IN ('Delhi', 'Mumbai', 'Pune');

-- LIKE (pattern match)
SELECT * FROM students WHERE name LIKE 'A%';   -- A से शुरू
SELECT * FROM students WHERE name LIKE '%kumar'; -- kumar पर end
SELECT * FROM students WHERE email LIKE '%@gmail%';

-- NULL check
SELECT * FROM students WHERE email IS NULL;
SELECT * FROM students WHERE email IS NOT NULL;
Ch 05

INSERT, UPDATE, DELETE

✏️ Data Modify करना
-- INSERT: Data डालना
INSERT INTO students (name, email, age, city)
VALUES ('Anshu', 'anshu@email.com', 20, 'Delhi');

-- Multiple rows insert
INSERT INTO students (name, age, city) VALUES
    ('Priya', 22, 'Mumbai'),
    ('Rahul', 21, 'Pune'),
    ('Sneha', 19, 'Delhi');

-- UPDATE: Data बदलना
UPDATE students SET age = 21 WHERE name = 'Anshu';
UPDATE students SET city = 'Bangalore', age = 23 WHERE id = 2;

-- DELETE: Data हटाना
DELETE FROM students WHERE id = 5;
-- ⚠️ WHERE नहीं दिया तो सारा data delete हो जाएगा!

⚠️ Golden Rule: UPDATE और DELETE में WHERE clause भूलना मत! बिना WHERE = सारा data affect होगा!

Ch 06

JOINs — Tables को जोड़ना

Real apps में data multiple tables में होता है। JOIN से tables combine करते हैं:

🔗 SQL JOINs
-- Tables setup
-- students: id, name, city
-- marks:    id, student_id, subject, score

-- INNER JOIN: दोनों tables में match हो
SELECT s.name, m.subject, m.score
FROM students s
INNER JOIN marks m ON s.id = m.student_id;

-- LEFT JOIN: Left table के सब rows + matched right
SELECT s.name, m.score
FROM students s
LEFT JOIN marks m ON s.id = m.student_id;
-- No marks? score = NULL

-- RIGHT JOIN: Right table के सब rows + matched left
SELECT s.name, m.score
FROM students s
RIGHT JOIN marks m ON s.id = m.student_id;
JOIN Type Result
INNER JOIN सिर्फ matching rows दोनों tables से
LEFT JOIN Left table के सब rows + right matched
RIGHT JOIN Right table के सब rows + left matched
FULL OUTER JOIN दोनों tables के सब rows
Ch 07

Aggregate Functions — GROUP BY

📊 Aggregation
-- Count
SELECT COUNT(*) AS total FROM students;

-- Sum, Average, Min, Max
SELECT
    SUM(score) AS total,
    AVG(score) AS average,
    MIN(score) AS minimum,
    MAX(score) AS maximum
FROM marks;

-- GROUP BY: city wise student count
SELECT city, COUNT(*) AS count
FROM students
GROUP BY city
ORDER BY count DESC;

-- HAVING: group filter (WHERE groups ke liye)
SELECT city, COUNT(*) AS cnt
FROM students
GROUP BY city
HAVING cnt > 5;

/*
 * ⭐ Execution Order:
 * FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
 */
Ch 08

Subqueries — Query के अंदर Query

🔄 Subqueries
-- Average से ज़्यादा marks वाले students
SELECT name, score FROM marks
WHERE score > (SELECT AVG(score) FROM marks);

-- Students जिन्होंने marks table में entry दी
SELECT * FROM students
WHERE id IN (SELECT DISTINCT student_id FROM marks);

-- EXISTS (check if subquery returns rows)
SELECT name FROM students s
WHERE EXISTS (
    SELECT 1 FROM marks m
    WHERE m.student_id = s.id
);
Ch 09

Indexes — Speed बढ़ाओ

⚡ Indexes
-- Index बनाना (search fast करता है)
CREATE INDEX idx_name ON students(name);
CREATE INDEX idx_city ON students(city);

-- Unique Index (duplicate values allowed नहीं)
CREATE UNIQUE INDEX idx_email ON students(email);

-- Indexes देखो
SHOW INDEX FROM students;

-- Index हटाओ
DROP INDEX idx_name ON students;

💡 Index कब लगाएं: WHERE, JOIN, ORDER BY में जो columns frequently आते हैं। लेकिन ज़्यादा indexes = slow INSERT/UPDATE!

Ch 10

Normalization — Data Organize करना

Normalization = data redundancy (दोहराव) कम करना और consistency बढ़ाना:

Normal Form Rule Example
1NF हर cell में single value skills = "HTML,CSS" ❌ → अलग rows बनाओ
2NF 1NF + No partial dependency Non-key columns पूरी primary key पर depend
3NF 2NF + No transitive dependency Non-key column दूसरे non-key पर depend न करे

💡 Practical Rule: ज़्यादातर apps 3NF enough है। Over-normalization से queries complex हो जाती हैं और performance गिर सकती है।

Ch 12

🎯 Interview Questions

Q1: WHERE vs HAVING difference?

→ WHERE rows filter करता है (GROUP BY से पहले)। HAVING groups filter करता है (GROUP BY के बाद)।

Q2: INNER JOIN vs LEFT JOIN?

→ INNER = सिर्फ matching rows। LEFT = left table के सब rows + right matched (no match = NULL)।

Q3: Primary Key vs Foreign Key?

→ Primary Key = unique identifier (NOT NULL, unique)। Foreign Key = दूसरी table की primary key reference।

Q4: Index क्यों use करते हैं?

→ SELECT queries fast करने के लिए। लेकिन INSERT/UPDATE slow हो सकते हैं। Trade-off!

Q5: DELETE vs TRUNCATE vs DROP?

→ DELETE = rows delete (WHERE possible, rollback possible)। TRUNCATE = सब rows delete (fast, no rollback)। DROP = पूरी table delete।
