Ch 01

Java क्या है? — Introduction

Java दुनिया की सबसे popular programming languages में से एक है। यह Object-Oriented, Platform Independent, और Secure है।

  • "Write Once, Run Anywhere" — JVM (Java Virtual Machine) की वजह से हर OS पर चलता है
  • Backend Development — Spring Boot, enterprise applications
  • Android Development — Android apps Java/Kotlin में बनती हैं
  • High Demand — India में सबसे ज़्यादा Java jobs हैं
  • Strong OOP — real-world problems को objects से solve करता है

💡 Java vs JavaScript: दोनों बिल्कुल अलग languages हैं! Java = compiled, strongly typed, OOP। JavaScript = interpreted, dynamic, web-focused।

Variables & Data Types

Java strongly typed है — हर variable का type पहले बताना पड़ता है:

📦 Data Types
// Primitive Types (basic)
int age = 20;            // Integer (whole number)
double price = 99.99;    // Decimal number
char grade = 'A';        // Single character
boolean isStudent = true; // true/false
float pi = 3.14f;        // Float (f ज़रूरी)
long bigNum = 999999999L; // Long (L ज़रूरी)

// Reference Type
String name = "Anshu";   // String (capital S!)

// Constants
final int MAX = 100;    // final = cannot change (like const)

// Type Casting
int x = (int) 9.7;     // 9 (explicit — data loss!)
double y = 5;           // 5.0 (implicit — safe)

Operators & Control Flow

🔀 Conditions & Loops
// If-Else
int marks = 85;
if (marks >= 90) {
    System.out.println("A+ Grade");
} else if (marks >= 80) {
    System.out.println("A Grade");
} else {
    System.out.println("Need improvement");
}

// For Loop
for (int i = 1; i <= 5; i++) {
    System.out.println("Count: " + i);
}

// For-Each (arrays/collections)
String[] fruits = {"Apple", "Banana", "Mango"};
for (String fruit : fruits) {
    System.out.println(fruit);
}

// While Loop
int count = 0;
while (count < 3) {
    System.out.println(count);
    count++;
}
Ch 05

OOP — Classes & Objects

Java Object-Oriented है — real world चीजों को code में represent करो:

🧱 Class & Object
// Class = Blueprint (design/template)
public class Student {
    // Properties (data)
    String name;
    int age;

    // Constructor (object बनाते समय call)
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Method (behavior/action)
    public void introduce() {
        System.out.println("Hi, I'm " + name + ", age " + age);
    }
}

// Object = Blueprint से बनी actual चीज
Student s1 = new Student("Anshu", 20);
s1.introduce();  // "Hi, I'm Anshu, age 20"

🔑 OOP 4 Pillars: Encapsulation (data hiding), Inheritance (reuse), Polymorphism (many forms), Abstraction (simplify)

Ch 06

Inheritance — Code Reuse

👨‍👦 Inheritance
// Parent Class
class Animal {
    String name;
    void eat() { System.out.println(name + " is eating"); }
}

// Child Class — extends keyword
class Dog extends Animal {
    void bark() { System.out.println("Woof! 🐕"); }
}

Dog d = new Dog();
d.name = "Buddy";
d.eat();   // Parent method — "Buddy is eating"
d.bark();  // Own method — "Woof! 🐕"
Ch 07

Polymorphism — Many Forms

🎭 Method Overloading & Overriding
// Method OVERLOADING (same name, different parameters)
class Calculator {
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }
    int add(int a, int b, int c) { return a+b+c; }
}

// Method OVERRIDING (child redefines parent method)
class Animal {
    void sound() { System.out.println("..."); }
}
class Cat extends Animal {
    @Override
    void sound() { System.out.println("Meow! 🐱"); }
}
Ch 08

Abstraction & Interface

🎨 Abstract Class & Interface
// Abstract Class — partial implementation
abstract class Shape {
    abstract double area();  // No body — child must implement
    void display() { System.out.println("I'm a shape"); }
}

class Circle extends Shape {
    double radius;
    Circle(double r) { this.radius = r; }
    double area() { return 3.14 * radius * radius; }
}

// Interface — 100% abstract (contract)
interface Drawable {
    void draw();  // must implement
}

class Square implements Drawable {
    public void draw() { System.out.println("Drawing □"); }
}

Exception Handling — Error पकड़ो

🛡️ Try-Catch
try {
    int result = 10 / 0;  // ArithmeticException!
} catch (ArithmeticException e) {
    System.out.println("Error: " + e.getMessage());
} finally {
    System.out.println("यह हमेशा चलेगा!");
}

// Custom Exception
class AgeException extends Exception {
    AgeException(String msg) { super(msg); }
}

// throws keyword
void checkAge(int age) throws AgeException {
    if (age < 18) throw new AgeException("Too young!");
}

Collections — Data Structures

📚 Collections Framework
import java.util.*;

// ArrayList — dynamic array
ArrayList<String> list = new ArrayList<>();
list.add("HTML");
list.add("CSS");
list.add("Java");

HashMap<String, Integer> map = new HashMap<>();

// HashSet — unique values only
HashSet<String> set = new HashSet<>();

set.add("Java");  // Duplicate — ignored!
set.size();       // 1

Ch 12

🎯 Interview Questions

Q1: JDK, JRE, JVM में फर्क?

→ JVM = code execute करता है। JRE = JVM + libraries। JDK = JRE + compiler + tools (development के लिए)।

Q2: Abstract Class vs Interface?

→ Abstract = partial implementation, single inheritance। Interface = 100% contract, multiple implement कर सकते हैं।

Q3: Overloading vs Overriding?

→ Overloading = same name, different params (compile-time)। Overriding = child redefines parent method (runtime)।

Q4: == vs .equals()?

→ == reference compare करता है (memory address)। .equals() value compare करता है (content)। Strings के लिए .equals() use करो!

Q5: ArrayList vs LinkedList?

→ ArrayList = fast random access, slow insert/delete। LinkedList = slow access, fast insert/delete। Most cases में ArrayList better।
